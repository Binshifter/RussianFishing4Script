# 俄罗斯钓鱼4全自动钓鱼脚本

</div>

![img.png](img.png)


#### 创建环境

```bash
python -m venv .venv
.venv\Scripts\activate
python.exe -m pip install --upgrade pip
pip install -r requirements.txt
```

## 说明

开启自动进食：脚本每隔约一段时间自动按一次游戏内的4键，需要把食物放在4键上。

达标重量倍数：AI判断(对接火山引擎)鱼获的筛选标准，填1表示只保留达到标准重量的鱼，0表示强制保留所有鱼入户，如果使用除了0外的其他数字，需要自己准备api调用，做这个功能是之前没有想好要怎么实现卖鱼，让ai对照着数据库判断改留下还是放生，增加满户的时间，如果单纯想要赚银币，不建议使用这个功能

收线模式：目前做了三种。平收、JIG模式。海钓模式。选择平收和JIG可以启用自动卖鱼，海钓目前还不支持，海图寻路比较复杂，目前还没有能力做到。

开启自动装配引线/拟饵：勾选后程序会每隔一段时间检查，主要是防止引线被咬断的情况，引线公斤数和拟饵型号会从模板文件夹自动扫描生成下拉列表，直接点选即可，没有的型号和引线可以自行截图、命名。

开启重登回城卖鱼：勾选后需要选择服务器类型和卖鱼地图点位。自动卖鱼我用到了小退然后执行两端固定脚本，一段是从登录点到卖鱼点的脚本类似于(XX_initial_to_sell.json这样的文件)，一段的从卖鱼点到各钓鱼点的脚本(XX_X_sell_to_fishing.json这样的文件)，目前只准备了2图和白河的脚本，主要是供搬砖使用，可以自行录入脚本，然后去map_config.json中仿照2图和白河补齐对应的点位就行了。

大鱼超时自动重登卖鱼：勾选后，如果连续多次收线超时，默认一次是1分钟，达到设定次数后自动执行重登卖鱼，防止被大鱼卡住或者损害鱼竿轮子位置。连续超时次数默认3次，就是这条鱼3分钟还没拉上来，就会执行重登卖鱼，可根据钓点情况调整。(这个功能好像还有点问题，不完善)

时间策略配置：用于防检测，没做这个功能之前连续运行了3天3夜喜提100公斤，建议每日运行不超过12小时

注意事项：
1、脚本启动前需要先配置好路亚，包括摩擦和转速
2、游戏窗口化运行，并且将游戏内控制中的的鼠标灵敏度调整至最低。
3、鼠标灵敏度调整至最低后，如果还是和我的2图和白河的重登脚本(去卖鱼点和去钓鱼点)对不上，可能需要去map_config.json中调整一下对应点位的initial_rotation与water_rotation的值。
4、不要在程序运行的过程中，特别是在重登操作时候去动鼠标，建议挂机运行的时候拔掉鼠标
